# Гайд разработчика IWE

> Для разработчиков T4+ (TD1). Если вы не знаете свой tier — сначала прочитайте `docs/LEARNING-PATH.md`.

## Конвейер разработки — одной страницей

Разработка в IWE проходит через **6 станций** с инвариантом **двойного выхода** (код + зафиксированное знание):

1. **Постановка** — сырая нужда → задача (тег маршрутизации, класс верификации, критерий приёмки).
2. **Открытие** — WP Gate: роль, работа, класс, оценка, модель. Согласование с пилотом обязательно.
3. **Проектирование** — IntegrationGate/ArchGate для непростого; для тривиального — skip.
4. **Работа** — код + capture одновременно (не "сначала код, потом документация").
5. **Проверка** — по классу верификации: closed-loop → чеклист/тесты; open-loop → peer-сессия; problem-framing → сверка с эталоном (R23/VR).
6. **Закрытие** — PR, merge ведущим разработчиком (TD1+TA4) или пилотом, обновление реестра.

**Двойной выход:** задача, после которой остался только код, считается **незакрытой**. Capture = различение, memory-файл, обновление Pack или AGENTS.md.

## Что делать с первой карточкой

1. Скопируй шаблон в свою папку задач: `cp docs/templates/conveyor-card.md <твоё-пространство>/inbox/tasks/my-card.md` (реестр и `inbox/tasks/` живут в твоём DS-пространстве, не в шаблоне).
2. Заполни frontmatter (wp, verification_class, estimate, double_exit)
3. Пройди 6 станций (карточка = вход для станции 1)
4. Закрытие: PR в репо + capture в distinctions/memory

## WP Gate — как открыть задачу

См. `CLAUDE.md §2 Pre-action Gates`. Обязательно объяви: роль, работу, РП, класс верификации, метод, оценку, модель. Дождись согласования пилота.

## Критерий «сделано»

- [ ] Код работает (или артефакт создан)
- [ ] Capture зафиксирован (различение / memory / Pack)
- [ ] РП закрыт в реестре (`<твоё-пространство>/docs/WP-REGISTRY.md`)
- [ ] PR влит (merge — ведущий разработчик TD1+TA4 или пилот)

## Pull Request — шаблон обязателен

При открытии Pull Request автоматически подставляется шаблон (`.github/PULL_REQUEST_TEMPLATE.md`): ссылка на карточку, двойной выход, чеклист 6 станций, класс верификации. Заполни его честно - ревьюер по нему видит, что конвейер пройден. Пустой чеклист = PR не принимается.

## Кто принимает merge

**Только** ведущий разработчик (TD1+TA4) или пилот. Никто другой — без явного делегирования.

## Режим отказа

Если задача застряла дольше оценки (closed-loop — часы, open-loop — дни) — эскалация ведущему разработчику или пилоту. Не буксуй тихо.
