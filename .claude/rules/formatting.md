# Правила форматирования

> Загружается всегда. Источник: CLAUDE.md § 6 (вынесено для сжатия контекста).

## Язык — русский (БЛОКИРУЮЩЕЕ)

Все ответы, метки, категории, заголовки — на русском. EN только в коде, терминах (DDD, BC, UL), именах файлов.
НЕ: HIGH / MEDIUM / LOW / CRITICAL / DONE. ДА: **критический**, **высокий**, **средний**, **низкий**, **выполнено**.

## Таблицы с РП

- Активные (pending, in_progress): название **жирным**
- Done: зачёркнуть строку без bold (`| ~~#~~ | ~~название~~ | ✅ | ~~done~~ |`). Эмодзи-статусы (✅, 🔄, ↗️, 📦) — **без** `~~`. Тильда `~` внутри `~~` ломает GitHub — заменять `~30 мин` → `30 мин`, `{{HOME_DIR}}/IWE` → `IWE`.

## Структура длинных документов (>80 строк)

Документ >80 строк → разделять на секции через `##` заголовки. Без HTML-тегов (`<details>`, `<summary>`) — Obsidian их не рендерит корректно (ломает таблицы внутри).

Применять к: WeekPlan, DayPlan, Strategy, WP-context files, спецификации.
